## Sécurité (OWASP appliqué)

- **A07 Authentication Failures** : `PAID_PLANS = {"basic"}` empêche les
  checkouts directs sur les plans legacy (anti-fraude : si l'UI ne propose
  plus ces plans, l'API les refuse aussi).
- **A07** : `register_user()` accepte uniquement les 4 langues supportées
  en écriture. Les comptes legacy avec `language='ar'` etc. restent
  lisibles (pas de migration forcée requise).
- **Information disclosure mitigation** : `reset_password()` retourne un
  message générique (`auth_invalid_token`) pour les 3 cas d'échec (token
  inconnu, déjà utilisé, expiré) afin de ne pas révéler l'état du token
  aux attaquants.
- **Input validation** : `parse_accept_language()` parse de manière
  defensive le header RFC 7231 (try/except sur `q=`), `normalizeLang()`
  whitelist côté mobile et backend.
